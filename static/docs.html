VSP P4 Compliance API

REST API for DoD Zero Trust P4 compliance management. Provides programmatic access to RMF workflow, Zero Trust scoring, compliance pipeline, SBOM, OSCAL export, and alert management.

Base URL http://127.0.0.1:8921 · Version 2.0.0 · OSCAL 1.1.2
Authentication
All P4 API endpoints (except /health) require authentication via Bearer token or X-API-Key header.
Request Headers
Authorization: Bearer <your-jwt-token> # OR X-API-Key: <your-api-key> # OR (dev/testing only) ?token=<your-jwt-token>
Get JWT Token
curl -X POST http://127.0.0.1:8921/api/v1/auth/login \ -H "Content-Type: application/json" \ -d '{"email":"admin@vsp.local","password":"your-password"}' # Response: {"token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."}
P4 Compliance
Core P4 compliance endpoints for Zero Trust scoring, RMF workflow, and pipeline management.
GET /api/p4/health Public health check — no auth required
Response
{"status":"ok","module":"VSP-P4","version":"2.0.0","p4_achieved":true}
200 OK
GET /api/p4/zt/status Full Zero Trust 7-pillar scorecardAuth
⚠ Requires authentication
Example Request
curl http://127.0.0.1:8921/api/p4/zt/status \ -H "Authorization: Bearer $TOKEN"
Response Fields
FieldTypeDescription
pillarsobjectAll 7 ZT pillar scores and capabilities
p4_readinessintOverall P4 readiness percentage (0-100)
p4_achievedboolTrue if all pillars ≥85 (P4 threshold)
overall_scoreintAverage score across all 7 pillars
sbomobjectSBOM summary (total, critical, high CVEs)
rasp_coveragearrayRASP deployment status per service
200 OK 401 Unauthorized
GET /api/p4/zt/microseg List micro-segmentation rulesAuth
⚠ Requires authentication
Add New Rule (POST)
curl -X POST http://127.0.0.1:8921/api/p4/zt/microseg \ -H "Authorization: Bearer $TOKEN" \ -H "Content-Type: application/json" \ -d '{ "name": "vsp-api → new-service", "source": "vsp-api", "destination": "new-service", "port": 8080, "protocol": "TCP", "action": "allow", "mtls": true }'
200 OK

⚠ Requires authentication

FieldTypeDescription
