## What went well

### Research-before-fix saved major time

Day 1 plan said merge fix/all-panels-token-bootstrap + codemod 140 sites. Research revealed:
- vsp_iframe_bootstrap.js v2 already superseded v1 attempt
- All 25 panels already on v2
- Backend cookie infra already 100 percent present

Result: ~5 days of work avoided. Cookie migration reframed from build infra to audit + flip flag.

### Classifier and codemod tools are reusable

Tools produced during Sprint 4:
- scripts/sprint4/classify-ui-debt.sh
- scripts/sprint4/classify-innerhtml-high.py
- scripts/sprint4/migrate-text-innerhtml.py
- scripts/sprint4/mark-structured-safe.py

Reusable for Sprint 5 frontend cleanup and Sprint 6 MED/LOW pass.

### Evidence-gated closure prevented a lie

Phase 3 initial commit claimed 5 sites migrated but Python patch block had been skipped during paste, only 1 site was actually in the diff. Classifier re-run caught this immediately. Amended commit before push, the lie never reached main or PR.

This is the meta-rule in docs/SECURITY_DECISIONS.md working as designed.

### Pre-commit hooks held the line

gofmt, go vet, and secrets check ran automatically on every commit. Zero secrets leaked.

## What went poorly

### Paste-Python-into-bash caused a 51MB stray file

Tried python3 heredoc with code containing import os, bash interpreter created a literal file named os (51 megabytes). Caught via git status.

Lesson: Always write Python to a file first, then run it. Never inline multi-line Python via heredoc.

### Heredoc content with backticks and em-dashes got truncated in terminal

Multiple attempts to write long markdown docs via cat heredoc failed silently, terminal buffer did not pass full content to bash. Fixed by writing files via Claude create_file tool and uploading directly.

Lesson: for any file longer than ~60 lines with special characters, write to a real file and upload, do not paste via heredoc.

### Estimation was consistently too pessimistic

Phase 1 TEXT estimated 3-4 hours, actual ~45 min. Cookie migration estimated 5-7 days, actual doc 30 min.

Why: Estimates based on touching N sites rather than understanding unique code paths.

Adjustment: Sprint 5 estimates should include mandatory classify-first phase.

### Pivoted 4 times during Sprint 4 kickoff

Iframe bootstrap strategy pivoted: codemod 144 sites, merge team branch, skip merge use v2, already done.

Lesson: For any migrate N sites task, grep top 3 files and head -30 each. Pattern emerges in 2 minutes.

### GitHub Actions CI still blocked

SD-0049 (billing outage) remains open. All Sprint 4 evidence came from local-ci.sh. Branch protection not enforceable until billing resolved.

User-side action required.
